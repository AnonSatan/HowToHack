Learning to Hack: The Methods

Packages

We'll start with software, known as "packages" in the linux world.

  • Fierce, a DNS sniffer.
  • nmap, a network scanner.
  • The Metasploit Framework.

Acquiring a Target

Target = domain.com. Now what?

We want to know all of the DNS entries associated with domain.com. We run Fierce and dump the results to a text file on the Desktop. 

perl /usr/bin/fierce -dns domain.com -file /root/Desktop/domain.com.txt

Opening domain.com.txt will reveal some IP addresses and their associated hostnames. We want to edit this text, leaving only the IP address and save this into a file called ips.domain.com.txt.

Scanning the Target(s)

Using the file ips.domain.com.txt, we run a scan using nmap. 

nmap -vvvv -A -iL /root/Desktop/ip.domain.com.txt -oX /root/Desktop/nmap.domain.com.xml

This string will run nmap with verbosity, OS/Version/script scanning and traceroute using the IPs listed in ips.domain.com.txt with an output file of nmap.domain.com.xml saved to the Desktop. The xml file will be used to import the results into the Metasploit Framework. Of course, you can save the files to any folder. I have found that some of the tools don't recognize using the ~/ for the home folder. So I have become accustomed to using the absolute paths.
