Learning to Hack: Metasploit Framework

Let's Get Serious

Now that you know about Fierce and nmap, let's bring this all together with the Metasploit Framework . You can view a Getting Started guide here. Metasploit currently has over 800 tools and scripts for network testing. But the best feature, in my opinion, is that it integrates with the Postgresql database. So all of the data from your scans can be stored and searched with relative ease.

To get started in the linux shell, issue the following commands: 

                service postgresql start
                service metasploit start

The first time metasploit is run, it will take a few minutes to initialize the databases it needs. Then run: 

                msfconsole

You are now in the metasploit shell.

I'm not going to go into the information provided in the Getting Started guide. I will offer up some tips on using Metasploit in console mode.

I make frequent use of the 

                workspace -a new.workspace.name

command. This will add a workspace by the name of the domain I'm working with, just to keep everything organized. Now, remember that file we made with nmap? We can import the .xml file into metasploit using the 

                db_import /path/to/xml/file

command. Now issue the 

                hosts

command to see the hosts that were imported into your workspace. If you now issue the 

                hosts -R

command, it will load all of the hosts that can now be used for whatever actions you may perform.
